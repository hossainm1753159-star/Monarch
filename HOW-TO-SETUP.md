# Monarch ওয়েবসাইট — সেটআপ গাইড (বাংলায়)

## কেন এই ধাপগুলো দরকার?
আগে আপনার সাইটে যে "Manage Store" দিয়ে প্রোডাক্ট পরিবর্তন করার ফিচার ছিল, সেটা শুধু Claude-এর ভেতরে কাজ করত। এখন সাইটটা GitHub-এ হোস্ট হচ্ছে, তাই প্রোডাক্ট/ছবি/নাম/দাম পরিবর্তন **সবার জন্য** সেভ রাখতে একটা ফ্রি ডেটাবেস (Firebase) লাগবে। এই গাইড অনুসরণ করলে ৫-১০ মিনিটেই এটা কাজ করবে।

এটা না করলেও সাইট চলবে, কিন্তু তখন আপনার পরিবর্তন শুধু আপনার নিজের ব্রাউজারেই দেখা যাবে — অন্য কেউ (কাস্টমার) দেখবে না।

---

## ধাপ ১: Firebase প্রজেক্ট তৈরি করুন (ফ্রি)
1. [console.firebase.google.com](https://console.firebase.google.com) এ যান এবং Google অ্যাকাউন্ট দিয়ে লগইন করুন।
2. **"Add project" / "Create a project"** ক্লিক করুন।
3. প্রজেক্টের নাম দিন, যেমন: `monarch-shop`। **Continue** চাপুন।
4. Google Analytics-এর অপশন আসবে — এটা বন্ধ (disable) রাখলেও চলবে। **Create project** চাপুন।

## ধাপ ২: Firestore ডেটাবেস চালু করুন
1. বাম পাশের মেনু থেকে **Build → Firestore Database** এ যান।
2. **"Create database"** চাপুন।
3. Location যেকোনো একটা কাছের রিজিয়ন সিলেক্ট করুন (যেমন `asia-south1`)। **Next** চাপুন।
4. **"Start in test mode"** সিলেক্ট করুন। **Enable/Create** চাপুন।

> ⚠️ Test mode-এ ৩০ দিন পর ডেটাবেস অটোমেটিক লক হয়ে যাবে (কেউ লিখতে পারবে না)। ৩০ দিনের মধ্যে নিচের "Rules" ট্যাব থেকে rules টা এভাবে বদলে দিন যাতে মেয়াদ শেষ না হয়:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```
(এটা "Publish" করে দিন)

## ধাপ ৩: Web App যোগ করুন এবং কনফিগ কপি করুন
1. প্রজেক্টের হোমপেজে (⚙ Settings আইকনের পাশে) **"</>"** (Web) আইকনে ক্লিক করুন।
2. অ্যাপের নিকনেম দিন, যেমন `monarch-web`। **"Also set up Firebase Hosting"** টিক দেওয়ার দরকার নেই। **Register app** চাপুন।
3. এখন একটা কোড ব্লক দেখাবে যেখানে `firebaseConfig = { apiKey: "...", ... }` লেখা আছে। পুরো অবজেক্টটা কপি করুন।

## ধাপ ৪: আপনার ওয়েবসাইটের ফাইলে বসান
1. এই ফোল্ডারে থাকা **`firebase-config.js`** ফাইলটা খুলুন (Notepad/TextEdit দিয়ে)।
2. `firebaseConfig = { ... }` এর ভেতরের placeholder মানগুলো (YOUR_API_KEY ইত্যাদি) মুছে Firebase থেকে কপি করা আসল মানগুলো বসান।
3. চাইলে `MONARCH_ADMIN_PASSWORD = "monarch123"` — এই পাসওয়ার্ডটাও নিজের ইচ্ছামতো বদলে দিন। এটা দিয়েই আপনি "Manage Store" চালু করবেন।
4. ফাইলটা সেভ করুন।

## ধাপ ৫: GitHub-এ আপলোড করুন
1. আপনার GitHub repository-তে যান।
2. **"Add file" → "Upload files"** চেপে আপডেট করা `firebase-config.js` ফাইলটা আপলোড করুন (আগেরটা রিপ্লেস হয়ে যাবে)।
3. **"Commit changes"** চাপুন।
4. ১-২ মিনিট পর আপনার লাইভ সাইট রিফ্রেশ করে দেখুন — এখন "⚙ Manage Store" বাটনে পাসওয়ার্ড চাইবে, আর যা পরিবর্তন করবেন তা **সব ভিজিটর** দেখতে পাবে।

---

## এরপর যা করতে পারবেন
- **⚙ Manage Store** বাটনে ক্লিক করে পাসওয়ার্ড দিন।
- প্রতিটা প্রোডাক্ট কার্ডে **✎ Edit** ও **🗑 Remove** বাটন আসবে — নাম, দাম, ছবি, ক্যাটাগরি, কালার, সাইজ, বর্ণনা সব বদলাতে পারবেন।
- **"+ Add Product"** কার্ডে ক্লিক করে নতুন প্রোডাক্ট (ছবি সহ) যোগ করতে পারবেন।
- হোমপেজের হেডলাইন এবং About পেজের লেখাও সরাসরি ক্লিক করে এডিট করতে পারবেন।

## সতর্কতা (নিরাপত্তা)
এই পাসওয়ার্ড সিস্টেমটা সাধারণ ভিজিটরদের ঠেকানোর জন্য যথেষ্ট, কিন্তু এটা কোনো ব্যাংক-লেভেল নিরাপত্তা না — টেকনিক্যাল কেউ চাইলে সরাসরি ডেটাবেসে লিখতে পারার সম্ভাবনা থাকে। ভবিষ্যতে আরও শক্ত নিরাপত্তা (Firebase Authentication) দরকার হলে জানাবেন, সেটা যোগ করে দেওয়া যাবে।
